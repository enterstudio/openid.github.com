Module Hierarchy

  • openid: This package is an implementation of the OpenID specification in Python.
    • openid.association: This module contains code for dealing with associations between consumers and servers.
    • openid.consumer: This package contains the portions of the library used only when implementing an OpenID consumer.
      • openid.consumer.consumer: OpenID support for Relying Parties (aka Consumers).
      • openid.consumer.discover: Functions to discover OpenID endpoints from identifiers.
      • openid.consumer.html_parse: This module implements a VERY limited parser that finds <link> tags in the head of HTML or XHTML documents and parses out their attributes according to the OpenID spec.
    • openid.cryptutil: Module containing a cryptographic-quality source of randomness and other cryptographically useful functionality
    • openid.dh
    • openid.extension
    • openid.extensions: OpenID Extension modules.
      • openid.extensions.ax: Implements the OpenID Attribute Exchange specification, version 1.0.
      • openid.extensions.draft
        • openid.extensions.draft.pape2: An implementation of the OpenID Provider Authentication Policy Extension 1.0
        • openid.extensions.draft.pape5: An implementation of the OpenID Provider Authentication Policy Extension 1.0, Draft 5
      • openid.extensions.sreg: Simple registration request and response parsing and object representation
    • openid.fetchers: This module contains the HTTP fetcher interface and several implementations.
    • openid.kvform
    • openid.message: Extension argument processing code
    • openid.oidutil: This module contains general utility code that is used throughout the library.
    • openid.server: This package contains the portions of the library used only when implementing an OpenID server.
      • openid.server.server: OpenID server protocol and logic.
      • openid.server.trustroot: This module contains the TrustRoot class, which helps handle trust root checking.
    • openid.sreg: moved to openid.extensions.sreg
    • openid.store: This package contains the modules related to this library's use of persistent storage.
      • openid.store.filestore: This module contains an OpenIDStore implementation backed by flat files.
      • openid.store.interface: This module contains the definition of the OpenIDStore interface.
      • openid.store.memstore: A simple store using only in-process memory.
      • openid.store.nonce
      • openid.store.sqlstore: This module contains OpenIDStore implementations that use various SQL databases to back them.
